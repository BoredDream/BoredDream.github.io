我的第二篇文章

发表于 2025-08-08 更新于 2026-01-21 分类于 博客搭建及diy 阅读时长 ≈ 1 分钟

准备工具：1.nodejs（自带npm）
2.git
3.hexo-cli

工具下载：1.node:https://nodejs.org
2.git:https://git-scm.com
3.hexo:npm install -g hexo-cli(cmd运行)

github仓库准备:
1.创建本名仓库.github.io
2.勾选”Initialize this repository with a README”
3.Creat

本地文件夹准备:
1.创建名为blog的空文件夹
2.在当前文件夹目录下运行

配置git并连接到github：
1.设置用户和用户邮箱：git config –global user.name “Github用户名(可以是别的名字)”
git config –global user.email “Github用户邮箱(也可以改别的)”
2.生成SSH密钥: ssh-keygen -t rsa -C “同上邮箱”
3.将密钥添加到github settings -> SSH keys
4.测试连接：ssh -T git@github.com
出现”hello,<github用户名>…”说明成功

修改hexo并配置文件_config.yaml：
1.找到deploy部分，修改为:
deploy:
type:git
repo:.github.io
branch:main
2.在blog目录下安装部署器依赖 –> npm install hexo -deployer-git –save

选择喜欢的主题(以volantis为例):
1.修改config.yaml中theme –>volantis
2.终端中输入–> npm i hexo-theme-volantis

主要几个指令:
1.hexo clean –>清除缓存文件(db.json)和public文件夹中已生成的静态文件(避免缓存干扰)
2.hexo g –>将markdown文件编译成静态文件并放到public文件夹
3.hexo s –>相当于hexo g生成了一个临时静态文件，然后可以实时在浏览本地博客，用于预览效果满意后再部署
4.hexo d –>将public文件夹中文件部署到Github
5.连招:hexo new post “title” –> hexo s(实时编写预览) –>hexo clean && hexo g -d(先生成静态文件后部署)
